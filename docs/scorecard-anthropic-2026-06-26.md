## Per-Dimension Detail

### 1. Pattern-fit & right-sizing — **B**
- **Anthropic principle:** "Building Effective Agents" — start with the simplest thing that works; add complexity only when it demonstrably improves outcomes; choose the pattern that fits (single call < chaining < routing < parallel < orchestrator-workers < evaluator-optimizer < autonomous); match model capability to task complexity; "delegate to subagents only when you need the result, not the journey."
- **Live evidence:** Orchestrator-workers done right — 11 agent files, 10 Haiku read-only researchers (Read/Grep/Glob/Bash, maxTurns 12-18), only `security-analyst` is Opus/maxTurns=25/effort=high. Build Prompt quotes the "Only delegate when you need the RESULT, not the journey" decision rule verbatim. `isComplexSession` gates sub-agents (predictedTimeout≥600 || correlated || kubernetes || flap/burst/multiple) so simple alerts stay single-agent. `handoff_depth` POLL≥5/halt≥10. Evaluator-optimizer gated (Should Review conf<0.7; Should Screen conf<0.6 || cost>€3). Unproven patterns default OFF (GEPA dormant); all 3 sentinels present. `team_formation.py` pure-rule (only `re`).
- **Gap to A:** (1) Biggest cost layer not right-sized — `session_log` shows opus-4-8:312 / opus-4-6:239 / opus-4-7:172, **zero sonnet across 738 real sessions**. The `modelHint='sonnet'` predicate is `isSimpleCategory && priorIncidents<=2 && !budgetExceeded`, but `priorIncidents=(kbRaw.match(/\|/g)||[]).length` counts **every pipe in the markdown-table KB output** → the `<=2` gate is blown past on any retrieval, so the route effectively never fires. (2) Build-ahead-of-need: `session_state_snapshot`=0 rows, `session_turns`=1 (a 2026-04-23 test). (3) `team_formation` charter is advisory text, not binding on delegation.
- **Fix:** Widen the predicate (drop pipe-counting; route availability/certificate/maintenance with conf-history>0.8 and no MIXED/HIGH band to Sonnet, keep Opus for correlated/kubernetes/storage/security/irreversible); plumbing already exists. **Effort low / Risk med / Orchestrator-dependent: yes.**

### 3. Modular design — **B**
- **Anthropic principle:** Composability — small single-responsibility tools, agents-as-needed with own context/allowlist, Skills as self-contained composable units, prompts as version-controlled single-source artifacts. Anti-pattern: the monolith mixing orchestration/config/instruction.
- **Live evidence:** Strong modular surfaces — 30 single-responsibility `scripts/lib` modules, 33 reusable `servers/*/client.py` wrappers, 11 subagents with proper `model:`/`tools:` frontmatter, 9 Skills with semver+allowed-tools+requires, centralized `config/` JSONs, `--settings config/dispatched-session-settings.json`. **Monolith:** Build Prompt Code node = 59,417 bytes / 523 lines, ~29 interpolated context blocks; the master `chatops-workflow` Skill is a hardcoded escaped JS string literal at L6 with a comment documenting a fragile two-place manual sync.
- **Gap to A:** **Single-source-of-truth violation is now LIVE** — inline literal still self-documents "12871 bytes" but live SKILL.md is 13,562 bytes (~691B stale, ~38B+ divergence). No automation guards the copy (`validate-n8n-code-nodes.sh` checks return-count only; holistic-health checks skill counts). Master skill = largest + least-modular block.
- **Fix:** Read SKILL.md body at build-time (upstream SSH `cat`/Read node → `$json.skillBody`); externalize sections into `config/prompt-sections/` with a thin renderer; add a CI divergence check. **Effort med / Risk med / Orchestrator-dependent: yes.**

### 5. Context management — **B**
- **Anthropic principle:** "Effective Context Engineering for AI Agents" (2025) — treat context as finite (context rot), curate smallest high-signal token set, compact at window boundaries, persist durable memory outside the window for cross-session continuity.
- **Live evidence:** Cross-session memory is the strongest part and LIVE — Stop+PreCompact+SubagentStop hooks wired; `mempal-session-save.sh` SAVE_INTERVAL=15 + loop-guard + 5-point save; `mempal-precompact.sh` always blocks; `compact-session-summary.py` cites "Anthropic Effective Context Engineering 2025"; hooks firing now. `session_transcripts`=19,141, `incident_knowledge`=1,697. Context editing: 6-signal RRF with diary, SOURCE_CAPS, compact-handoff-history.
- **Gap to A:** (1) Handoff-depth/cycle gate is DARK — `handoff_depth.py` well-built (BEGIN IMMEDIATE, POLL≥5/HALT≥10, cycle detection) but 0/75 sessions ever bumped, `handoff_log`=0, `write-handoff-metrics.sh` uncronned, no `handoff.prom`. (2) Per-turn telemetry dark for months (`session_state_snapshot`=0, `session_turns`=1); no per-session/daily token ceiling (only --max-turns 15 + €5 cost cap). (3) Response cap is a scatter — substring(0,4000)×26 + substring(0,60000)×2, no `RESPONSE_CAP` constant.
- **Fix:** Cron handoff metrics + synthetic handoff canary to exercise the gate; per-session token ceiling (totals already captured in Prepare Result) → fail-closed [POLL]+SMS; hoist response cap into one asserted constant. **Effort med / Risk low / Orchestrator-dependent: yes.**

### 6. Multi-agent coordination — **C**

- **Gap to A:** (a) `mine-failures-to-evals.py` **does not exist** — its `35 4 * * 0` cron is `[ -f ]`-guarded and silently no-ops → failure→eval flywheel is dead. ### 9. Resource/token economics — **B**

- **Live evidence:** Wired ceilings, degrade-not-deny — `MAX_DAILY=${MAX_DAILY_COST_USD:-25}` + `MAX_SESSION:-10` → BUDGET_EXCEEDED → `planFlag=' --plan'` (PLAN-ONLY); every Launch Claude wraps `timeout $TIMEOUT` (300/450/600); `handoff_depth` POLL≥5/halt≥10; `agent_as_tool` max_turns=15; SEARCH_BUDGET_S=10. finops-agora bounds batch jobs rigorously (`systemd-run -p MemoryMax -p MemorySwapMax=0 -p CPUQuota`, agora-batch.slice 14G, chaos rc=137 containment proof). - **Gap to A:** (1) The runner that spends 98.7% of the money uses `--scope --slice=app.slice` with **zero `-p MemoryMax`** (app.slice MemoryMax=infinity); no `claude-sessions.slice`. (2) No token ceiling — gate + `ChatOpsCostBudgetHigh` alert key on dollars, but Tier-2=$0 on Max sub so the real constraint (usage limits) is untracked. (3) Routing rarely downgrades (1 sonnet vs 440 opus/7d). (4) Gate keys on lagging reconciled `session_log.cost_usd` and only degrades to plan-mode, so the $361/day cascade ran 14× over the $25 gate, scaling with VOLUME not value. *(Alert is `ChatOpsCostBudgetHigh`, not `LLMSpendApproachingDailyGate`.)*

## Overall Grade — **B (B+ leaning)**

**Distribution:** 1 A · 7 B · 1 C · 1 D → **1 at A, 9 below A** (of 10 dimensions).

The system's load-bearing safety machinery (predict-before-act fail-closed gate, never-auto floor, territory hook) is genuinely A-grade and live-discriminating, and its modular/observability/eval/memory cores are real and well-engineered (B). The grade is dragged down by one structural theme that recurs across the lowest dimensions: **the orchestrator is effectively pinned to Opus** (D in Model selection, the #1 gap in Pattern-fit and Resource economics) because the cost-aware routing predicate is unreachable and the wiring is git-uncommitted — so "match model to task complexity" lands only at the cheap Haiku sub-agent tier while 98.7% of spend stays on Opus. The two highest-ROI fixes (#1, #2) attack exactly this and are both low-effort with the plumbing already in place. The other recurring B-capping pattern is **built-but-dark telemetry** (handoff_log, team_charter, session_state_snapshot, trace-metrics cron, mine-failures script) — capability shipped and QA-passed but never wired to fire, which is why Multi-agent sits at C. Closing the dark-component wiring (backlog #4–#6) plus the two model-routing fixes would credibly move the overall to A−.
