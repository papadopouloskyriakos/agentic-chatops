## Bugs caught + fixed during onboarding (worth knowing)

1. **`netplan apply` hangs on systemd-networkd "Could not enumerate links: Connection timed out"** once eBGP+full-IPv6-table is loaded — 241k+ kernel routes overflow the netlink window. **Reboot** to apply netplan changes; never `netplan apply` online. Memory: `feedback_netplan_apply_post_ebgp_blows_netlink_timeout.md`.
2. **iFog FastNetMon null-routes per-command SSH bursts** — defra01agri01 was null-routed twice 2026-04-27. Mandatory ControlMaster on all chaos-driver and helper SSH calls. Memory: `feedback_ssh_multiplex_for_ddos_safety.md`.
3. **HAProxy backend `ssl verify required` needed for Hugo-with-TLS targets** — omoikane_www/_beta backends on the dmz pair serve nginx-with-TLS, plain-HTTP from HAProxy returns 400 → client sees ERR_HTTP2_SERVER_REFUSED_STREAM. Match the existing cubeos/mulecube/withelli/meshsat backend pattern. Memory: `feedback_haproxy_backend_tls_required_for_hugo_nginx_targets.md`.
4. **chaos catalog drift on every topology change** — `experiments/catalog.yaml` had pre-existing rtr01 gap since 2026-04-21; my TX additions tripped the validator and surfaced the drift. Always run `python3 scripts/chaos_catalog.py validate` after IaC topology changes. Memory: `feedback_chaos_catalog_drift_per_topology_change.md`.
5. **Mesh-stats helper `ssh_vps_swanctl()` doesn't multiplex** — when iFog null-routes claude01 → txhou, the helper returns empty dict. Workaround: read TX tunnel state from peer-side swanctl (chzrh + notrf01vps01 + dmz pair conn name `txhou01`). Live `vpn-mesh-stats.py` queries each peer instead of txhou directly.
6. **OS reinstall changes the SSH host key** — clear `~/.ssh/known_hosts` for both IP and hostname before reconnecting after a reinstall.
7. **CrowdSec listen_addr default `127.0.0.1`** — needed change to `0.0.0.0` for Prometheus to scrape `:6060` from the cluster. chzrh + notrf01vps01 already have this; txhou's fresh install defaulted to localhost.
